## Follow-ups noted but not turned into plans this pass

- **LazyMotion migration is still partial after 007**: ~20 more
  `use-lazy-motion` sites remain (`Disclosure.tsx` — still imports full
  `motion` for its chevron even after 009's CSS-reveal fix, `CorrectionList.tsx`,
  `TranscriptReview.tsx`, `ConverseTab.tsx`, `WeaknessList.tsx`,
  and others). The full ~30kb bundle reduction this rule promises needs a
  further pass migrating those too.
- **Optimistic queue advance in `grade()`** — advancing the queue immediately
  and reconciling `recordReview`/`getCounts` in the background would remove
  the highest-frequency perceived-latency moment in the app. Not written up
  as a plan since it changes user-visible timing/rollback behavior and
  deserves its own design discussion, not a mechanical fix.
- **Suspense boundary alongside 002's error boundary**, so a slow tab doesn't
  block the whole shell.
- **A shared `useAnnounce`/`LiveRegion` primitive** for the ad hoc
  `aria-live` gaps found in the Accessibility audit (`TranscriptReview.tsx`,
  `JsonImportForm.tsx`).
- Findings #6-#14 from the full audit table (Modal focus trap, AudioPlayer
  keyboard scrub, layout-property animations in `Disclosure`/`CorrectionList`/
  `TranscriptReview`, hydration-mismatch date formatting, label wiring gaps,
  the `src/features/plan/` dead-code island, unused store exports,
  `DiscoverTab`'s size) were vetted and presented but not selected for a plan
  this pass.
